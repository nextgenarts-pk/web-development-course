## 🎯 Week Overview

This week you'll learn **MongoDB** — the most popular NoSQL database — and **Mongoose**, the elegant ODM (Object Data Modeling) library for Node.js. You'll replace in-memory data stores with a real database, learn schema design, data validation, querying, and relationships.

---

## 📋 Learning Objectives

By the end of this week, you will:
- Understand NoSQL vs SQL databases and when to use MongoDB
- Set up MongoDB locally and with MongoDB Atlas (cloud)
- Design schemas and models with Mongoose
- Perform CRUD operations with Mongoose queries
- Implement data validation, indexes, and virtual fields
- Model relationships with references and population
- Build a complete **Database-Driven Notes App**

## 🔑 Key Concepts

- **Document Database** — Data stored as flexible JSON-like documents (BSON)
- **Collections** — Groups of documents (like SQL tables)
- **Schema** — Structure definition with types, validation, defaults
- **Model** — Constructor compiled from a Schema; interface to the database
- **ObjectId** — MongoDB's unique identifier for documents
- **Population** — Mongoose feature to replace IDs with actual documents
- **Indexing** — Database optimization for fast queries
- **Aggregation** — Pipeline for complex data transformations
